1) O elemento audio permite como elemento-filho o elemento source. Quais são  os atributos que o source permite e qual conteúdo eles suportam? 
Quais os  outros atributos do elemento audio e quando devem ser utilizados? 

RESPOSTA: O elemento source é utilizado para especificar múltiplos recursos de mídia de elementos audio e video em HTML5. É um elemento vazio.
	  Ele permite incluir vários formatos como tipos de alternativa caso o navegador do usuário não seja compatível com um deles. 
	  Outros atributos são o autoplay, que possiblita que o audio toque sem que o usuario seja solicitado. O preload, onde o autor indica ao
	  browser como vai ser reproduzido de acordo com o que o autor achar melhor.

3) No caso do canvas, HTML 5 nunca, em momento algum, fez qualquer  animação. Isso porquê o HTML 5 é uma simples linguagem de marcação. 
Dessa  forma, o que é necessário para criar animações com HTML5? O que são APIs do  HTML5?

RESPOSTA: Canvas é um elemento da HTML5 destinado a delimitar uma área para renderização dinâmica de gráficos. 
	  Todo o trabalho de criação e animação é realizado atráves de linguagens de programação dinâmica (usualmente Javascript) e a 
	  utilização de APIs é um conjunto de funcionalidades que não estão presentes em HTML, normalmente são escritas em Javascript e são 
	  importadas para os projetos.
